## Cómo trabajo

Me interesa que el código sea útil, comprensible y mantenible. En proyectos educativos priorizo una progresión clara: empezar con fundamentos sólidos, añadir complejidad de forma razonada y terminar con algo que se pueda ejecutar, probar y explicar.

En proyectos de producto busco una arquitectura sencilla, documentación suficiente y una experiencia práctica para quien tiene que usar o mantener la solución.
